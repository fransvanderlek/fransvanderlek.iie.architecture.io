RAMI4.0 : Architecture View - RobotArm Manifest ()
The manifest view shows what interfaces, components and behaviors are exposed by the I4.0 Component to it's environment. It serves as a contract whereby the I4.0 Component author or vendor will keep the manifest stable. Users of the I4.0 Component should only use artefacts and relationships shown in the manifest. Details can be found in the I4.0 Component Implementation view, though any artefacts on that view should be considerd private and never to be refered from outside, unless they appear in the manifest view as well. This division also makes manageable the creation of inferred relations between artefacts of the I4.0 Component, as currently tool support for such features is limited and authors should put those relations by hand. In general, the deferred relations are useful for overviews that do not need the specific details.
